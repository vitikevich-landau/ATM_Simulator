# Анимированная сцена дашборда — архитектура (feature/scene)

Как из дискретных снимков движка получается «живая картинка»: банкомат с
экраном, очередь человечков, приходы/уходы, анимации. Дизайн выбран на
этапе проработки (подход «Театр»: презентационный слой актёров + diff-рендер),
бюджет кадра снят инструментально — [scene_frame_budget.md](scene_frame_budget.md).

## Слои и поток данных

```
 AtmEngine (поток обслуживания, писатель)
    │  const-снимки под коротким локом: snapshot() / queueSnapshot() /
    │  statsSnapshot() / operations()          ← движок НЕ ИЗМЕНЁН вообще
    ▼
 renderLoop (render-поток LiveRenderer, частота кадров ui.scene_fps)
    │  снимки опрашиваются реже — на ui.refresh_hz (кэш в LiveRenderer)
    ▼
 ScenePresenter::tick(снимки, nowSec, лента операций)
    │  reconciliation: реестр актёров (по ClientId) догоняет снимок
    │  твины позиций, стадии жизни, судьбы ушедших, idle-фазы
    ▼
 SceneView («что рисовать»: актёры с позициями/позами + экран банкомата)
    ▼
 SceneComposer::composeScene (чистая функция)
    │  спрайты SceneSprites, глифы под контролем GlyphSet
    ▼
 SceneCanvas (клетки глиф+цвет) → toLines(color) → строки кадра
    ▼
 composeLinesFrom: шапка + СЦЕНА + полная таблица (очередь/статистика/лента)
    ▼
 FrameDiffer::diff — в терминал уходят только ИЗМЕНИВШИЕСЯ строки,
 кадр обёрнут в synchronized output (DEC 2026)
```

## Ключевые инварианты

* **Рендер только читает движок** (§4.8): весь новый код живёт в console-слое;
  presenter, differ и кэши снимков принадлежат render-потоку. Консоль общается
  с ними единственным атомарным флагом (`resume()` → полный repaint + телепорт).
* **Высота кадра постоянна в пределах live-сессии** (§4.8.5): «влезает ли
  сцена» решается один раз в конструкторе LiveRenderer (нужно ~84×30; иначе
  молча остаёмся в таблице). Команда `scene on|off` меняет режим ЧЕРЕЗ
  перезапуск live-сессии — тем же путём, что `live off` → `live`.
* **1 кодовая точка = 1 колонка**: сцена рисует только глифы из белого списка
  `GlyphSet` (ASCII, кириллица, box-drawing, блоки; НИКАКИХ эмодзи). В debug
  запрещённый глиф роняет assert прямо в `SceneCanvas::put()`.
* **Два времени.** Рендер-часы (steady_clock) двигают только косметику:
  походку, переминания, взмахи, эффекты. Смысл — этап обслуживания и его
  прогресс — приходит из снимка (`currentStage`, `serviceProgress`), поэтому
  пауза движка и любой `time_scale` обрабатываются конструкцией: на паузе
  человечки живут, но ничего смыслового не происходит.
* **Детерминизм без RNG**: личные фазы idle-анимаций и редкие взмахи —
  `splitmix64(clientId, номер окна)`. Время в `tick()` подаётся параметром.
  Одна последовательность снимков и времени ⇒ побайтно одинаковые кадры.

## Reconciliation (ScenePresenter)

Каждый tick — дифф множеств (события между кадрами коалесцируются сами):

| В снимке | В реестре | Действие |
|---|---|---|
| есть | нет | walk-in с правого края (телепорт на первом кадре/после resume) |
| есть, слот сменился | есть | новый твин ИЗ ТЕКУЩЕЙ точки, catch-up ≤0.7 c; телепорт при дистанции > полусцены |
| нет | есть | LeavePending: до 2 тиков ждём запись в ленте операций → OK: уходит зелёным `\o/` · FAIL: жёлтым «#id?» · нет записи: красным (не дождался) |
| хвост за видимыми слотами | — | не спавнится (виртуализация, счётчик «… ещё N») |

Одновременных уходящих ≤ 4 (старейший вытесняется). Кадр ног походки
выбирается по ПОЗИЦИИ актёра — темп шагов совпадает со скоростью движения.

## Конфигурация (`ui`)

| Ключ | Дефолт | Смысл |
|---|---|---|
| `scene` | `false` | сцена над таблицей (терминал от ~84×30; рекомендовано 120×40 — в Windows Terminal профиль `startingColumns`/`startingRows`) |
| `scene_fps` | `15` | частота кадров сцены (5..30) |
| `scene_rows` | `10` | высота полосы (10..14) |
| `scene_effects` | `true` | спиннер связи, купюры у лотка, «пар» злости |

Команды: `scene [on|off]` (в live — с перезапуском сессии), остальное как было.

## Что осталось на v2 (осознанно срезано)

* `scene_ascii_only` — фолбэк для растрового шрифта legacy conhost
  (box-drawing → `+-|`): целевая среда — Windows Terminal, а деградация в
  conhost с TrueType-шрифтом уже работает.
* Реакция на ресайз терминала на лету (сейчас размеры читаются один раз на
  live-сессию; `live off` → `live` пересчитывает).
* Клеточный (а не строчный) FrameDiffer — строчного диффа хватает с запасом
  (~1 мс на кадр в conhost).
* Truecolor-палитра по `WT_SESSION`.
