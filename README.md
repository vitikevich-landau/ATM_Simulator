# ATM Queue Simulator

Консольный симулятор работы банкомата на C++20: одноканальная система массового
обслуживания (СМО) — один банкомат, очередь клиентов со случайным временем
обслуживания, уходом по терпению и режимом технического обслуживания. Отдельно —
**неблокирующий** интерфейс администратора и **живая визуализация** процесса
прямо в консоли (Live-режим).

Полное техническое задание: [ATM_Simulator_TZ.md](ATM_Simulator_TZ.md).

| Этап | Что делает | Статус |
|------|------------|--------|
| **M0** | Каркас: структура, CMake, сборка и запуск на обеих ОС | ✅ |
| **M1** | Доменное ядро: `Money`, `Account`, `Cashbox`, `Operation`, `Client` + юнит-тесты | ✅ |
| **M2** | Конфигурация (JSON) + однопоточная симуляция очереди со статистикой СМО | ✅ |
| **M3** | Многопоточность: `AtmEngine` в `jthread`, прерываемое ожидание (§6.2), консоль (`status`/`queue`/`pause`/`resume`/`stop`) | ✅ |

| **M5** | Режим техобслуживания: `maintenance start/stop`, логика «уйти/остаться» (§4.5), авто-завершение | ✅ текущий |
| **M6** | Живой дашборд (Live-режим, §4.8) | ⏳ |
| **M7** | Логирование в файл, санитайзеры, документация, полировка | ⏳ |

## Сборка

### Вариант 1. Локально через CMake (штатный способ из ТЗ)

Нужны CMake ≥ 3.20 и компилятор с поддержкой C++20 (MSVC 19.3x, GCC ≥ 11, Clang ≥ 14).

```bash
cmake -S . -B build
cmake --build build
```

Запуск:

```bash
./build/atm_sim            # Linux
build\Debug\atm_sim.exe    # Windows (MSVC)
```

### Вариант 2. Проверка в Docker (используется при разработке)

На машине разработки сейчас нет локального C++-тулчейна, поэтому сборка и
проверка идут в Linux-контейнере с GCC 14 (`php:8.3-cli` — в нём уже есть
`g++`/`make`). Пример прямой компиляции (без CMake) для быстрой проверки:

```bash
docker run --rm -v "$PWD":/work -w /work php:8.3-cli \
  g++ -std=c++20 -O2 -Wall -Wextra -pthread -Iinclude src/main.cpp -o build/atm_sim
```

Тот же контейнер используется для прогона с **ThreadSanitizer** (поиск гонок
данных, §11/§14) — конкурентные тесты `AtmEngine`:

```bash
docker run --rm --security-opt seccomp=unconfined -v "$PWD":/work -w /work \
  php:8.3-cli sh scripts/tsan_docker.sh
```

> **Примечание по зависимостям.** По ТЗ конфигурация читается через
> `nlohmann/json`, а тесты — на GoogleTest/Catch2. Пока окружение разработки без
> доступа к сети, на первых этапах используется минимальный собственный
> тест-харнес (`tests/`), а внешние библиотеки подключаются через FetchContent,
> как только появится сеть. На результат симуляции это не влияет.

## Структура репозитория

```
atm-simulator/
├── CMakeLists.txt          # корневой скрипт сборки
├── config/
│   └── default_config.json # параметры симуляции (см. §9 ТЗ)
├── include/atmsim/         # публичные заголовки (core/ engine/ console/ …)
├── src/                    # реализация, зеркалирует include/
├── tests/                  # юнит- и сценарные тесты (с M1)
└── ATM_Simulator_TZ.md     # техническое задание
```

## Соглашения

- Идентификаторы кода — на английском; комментарии и документация — на русском
  (§12 ТЗ). Критические места (синхронизация, прерываемое ожидание) снабжаются
  подробными комментариями.
- Денежные суммы хранятся в **целочисленных** минимальных единицах валюты
  (копейки/центы), а не в `float`/`double` (§4.2 ТЗ).
